# Team Lead 任务包模板 / Task Packet Template

```md
# Task Packet

## 1. 任务名称

{一句话任务名}

## 2. 任务目标

{本轮只做什么}

## 3. 任务类型

- [ ] 新主题定义
- [ ] 内容收集
- [ ] 视频收集
- [ ] 图片生产
- [ ] 前端整合
- [ ] 纠错 / 审核

## 4. 适用主题

{knowledge-base / training-manual / hall-of-fame / other}

## 5. 必读文件

1. {framework}
2. {capability guide}
3. {state file if needed}

## 6. 输入

{已有 registry / 参考资料 / 批次范围}

## 7. 输出

### 必须输出

- {draft registry path}
- {report path}

### 不要直接修改

- {runtime file paths}

## 8. 范围限制

{本轮只处理哪些条目 / 模块 / 页面}

## 9. 质量门槛

- {最低标准 1}
- {最低标准 2}
- {最低标准 3}

## 10. 停止条件

- 完成 {X} 个条目后停止
- 遇到 {某类问题} 停止并报告
- 不要继续推进下一批

## 11. 已知风险

{可能缺资料、可能字段不一致、可能需要人工确认的点}

## 12. 完成后的汇报格式

1. 完成了什么
2. 输出了哪些文件
3. 哪些跳过
4. 哪些待确认
5. 下一步建议
```

---

## 模板使用原则

1. 每次任务包只服务一个能力角色。
2. 任务包必须小到可以一次完成或一次暂停。
3. 任务包优先限定范围，而不是鼓励穷尽整个世界。
