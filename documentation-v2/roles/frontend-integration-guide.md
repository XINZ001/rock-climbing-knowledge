# 前端整合指南 V2 / Frontend Integration Guide

> **能力定位**：把经过 review 的 draft 内容整合到前端运行时数据和展示层。

---

## 一、输入

1. 当前主题 framework
2. 当前 task packet
3. 通过 review 的 draft registry
4. 现有 runtime data

---

## 二、输出

1. 更新后的 runtime registry
2. 组件 / 页面调整
3. 整合报告

---

## 三、工作原则

1. 只接收已经过 review 的 draft
2. 优先复用现有组件和页面模式
3. 主题新增优先新增 registry 和 page type，不优先重写整站
4. 整合时记录字段映射规则

---

## 四、在不同主题下的工作区别

### 知识库

- 整合到 section / kp 页面

### 训练手册

- 整合到训练卡页面或训练卡模块

### 名人堂

- 整合到 profile 列表页和详情页

---

## 五、最关键的边界

前端整合不是内容研究。  
如果 draft 本身缺字段、缺结论、缺来源，应该退回上一层，而不是在整合层补脑。
