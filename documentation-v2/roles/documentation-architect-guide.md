# 文档架构负责人指南 V2 / Documentation Architect Guide

## 一、职责

维护文档体系，而不是维护某一条内容。

---

## 二、你要做的事

1. 保证文档类型清晰
2. 保证路径和真源一致
3. 保证 guide 可执行
4. 保证 framework 不掺状态
5. 保证 report 易复盘

---

## 三、你的判断原则

- 会变化的事实进 state
- 主题规则进 framework
- 角色执行进 guide
- 任务细节进 task packet

---

## 四、你优先修什么

1. 旧路径
2. 旧数字
3. 输出不明确
4. 停止条件不明确
5. guide 过长
