# 纠错员指南 V2 / Fact Checker Guide

> **能力定位**：处理争议、返工、错误修正，并为上游文档和 runtime data 提供修订建议。

---

## 一、输入

1. 当前主题 framework
2. 当前 task packet 或用户反馈
3. 涉及的 runtime data / draft data

---

## 二、输出

1. 纠错报告
2. 修改建议
3. 如获批准，再执行定点修正

---

## 三、工作原则

1. 先核实，再修改
2. 先报告，再执行
3. 只改明确确认的字段
4. 不顺手改无关内容

---

## 四、它在新体系中的位置

纠错不是知识库专属能力。  
知识库、训练手册、名人堂都可能调用同一套纠错流程。
