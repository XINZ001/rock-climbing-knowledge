# Team Lead 指南 V2 / Team Lead Guide

## 一、你的核心职责

你不是“写一份大说明书的人”，而是“给每轮执行发任务包的人”。

---

## 二、你的标准产出

1. framework
2. task packet
3. review decision
4. integration handoff

---

## 三、你的工作流

1. 判断这是新主题还是已有主题批次
2. 如果是新主题，先补 framework
3. 选择本轮能力角色
4. 生成一个小型 task packet
5. 等待执行结果
6. 审核是否通过
7. 决定是否进入整合层

---

## 四、不要做的事

- 不要把一个大 guide 原封不动扔给执行 bot
- 不要在一个任务包里同时塞多个能力角色
- 不要让研究类 bot 直接写 runtime data
